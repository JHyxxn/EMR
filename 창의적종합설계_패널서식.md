## 2. 설계 주제

**주제 선정 동기, 목적 등**

현대 의료 환경에서 대부분의 의료기관이 이미 EMR 시스템을 사용하고 있으나, 기존 상용 EMR 시스템들은 복잡한 인터페이스와 높은 도입 비용으로 인해 소규모 의료기관의 접근성이 낮은 실정입니다. 특히 AI 기술을 활용한 의료진 보조 기능이 부족하여, 의료진의 업무 효율성을 높이면서도 접근성이 좋은 차세대 EMR 시스템의 개발이 필요합니다.

**주제 선정 동기:**
- 기존 EMR 시스템의 복잡한 인터페이스로 인한 업무 효율성 저하 문제
- 반복적인 문서 작성 작업으로 인한 의료진의 업무 부담
- 약물 상호작용, 검사 결과 이상 수치를 놓칠 수 있는 의료 오류 위험
- AI 기반 보조 기능 부족으로 인한 자동화 기능의 한계

**목적:**
AI 기술을 활용하여 의료진의 진료 업무를 보조하고, 환자 정보를 효율적으로 관리할 수 있는 웹 기반 EMR 시스템을 개발하여 의료진의 업무 부담을 줄이고, 환자 안전을 향상시키며, 의료 데이터의 통합 관리를 실현하고자 합니다.

## 3. 설계 과정 설명

**기획, 설계, 제작 등 / 과정별 사진 활용**

**1단계: 기획 및 설계 (1주차)**
- 프로젝트 주제 선정 및 요구사항 분석
- 기술 스택 선정 (React, Node.js, Prisma, AI Gateway)
- 시스템 아키텍처 설계 (3-Tier 구조)
- 데이터베이스 스키마 설계

**[1단계 사진 삽입 위치: 시스템 아키텍처 다이어그램 또는 설계 회의 사진]**

**2단계: 백엔드 개발 (2-3주차)**
- Express.js 서버 구축 및 RESTful API 개발
- Prisma ORM을 통한 데이터베이스 연동
- 환자, 진료기록, 검사, 처방 관리 API 개발
- 한국 식품의약품안전처 DUR API를 활용한 약물 데이터베이스 구축
- 약물 상호작용 검사 시스템 개발

**[2단계 사진 삽입 위치: 백엔드 개발 화면 또는 API 테스트 화면]**

**3단계: AI Gateway 개발 (3-4주차)**
- 다중 AI 모델 통합 게이트웨이 구축
- 자동 폴백 메커니즘 구현
- 임상노트 요약, 증상 분석, 처방 가이드 등 AI 기능 개발

**[3단계 사진 삽입 위치: AI Gateway 구조도 또는 개발 화면]**

**4단계: 프론트엔드 개발 (4-6주차)**
- React + TypeScript 기반 UI 개발
- 통합 대시보드 구현
- 환자 등록, 진료 차트, 검사 관리, 문서 관리 페이지 개발
- API 연동 및 상태 관리 구현

**[4단계 사진 삽입 위치: 프론트엔드 개발 화면 또는 UI 컴포넌트]**

**5단계: 통합 및 테스트 (7-8주차)**
- 프론트엔드-백엔드-AI Gateway 통합 테스트
- 시뮬레이션 데이터 생성 및 테스트
