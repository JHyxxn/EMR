# 창의적 종합설계 결과보고서

## 1. 과제의 목적, 개발동기 및 필요성

### 1-1 캡스톤디자인 과제의 개요, 제작목적, 개발동기 및 필요성

현대 의료 환경에서 전자 의무기록(Electronic Medical Records, EMR) 시스템은 의료진의 업무 효율성과 환자 안전을 보장하는 핵심 인프라입니다. 기존 종이 기반 의무기록의 한계를 극복하고, 디지털화된 의료 정보 관리의 필요성이 대두되고 있습니다. 특히 소규모 의료기관에서는 고가의 상용 EMR 시스템 도입이 어려운 실정이며, 의료진의 업무 효율성을 높이면서도 접근성이 좋은 EMR 시스템의 개발이 필요합니다.

본 프로젝트는 AI 기술을 활용하여 의료진의 진료 업무를 보조하고, 환자 정보를 효율적으로 관리할 수 있는 웹 기반 EMR 시스템을 개발하는 것을 목표로 합니다. 이를 통해 의료진의 업무 부담을 줄이고, 환자 안전을 향상시키며, 의료 데이터의 통합 관리를 실현하고자 합니다.

**제작목적:**
1. 의료진 업무 효율성 향상: 환자 정보 검색, 진료 기록 작성, 처방 및 검사 오더 관리 등의 업무를 효율적으로 수행할 수 있는 통합 대시보드 제공
2. AI 기반 의료진 보조 기능 구현: 임상노트 자동 요약, 증상 분석 및 진단 추천, 약물 상호작용 검사, 검사 결과 이상 수치 자동 감지
3. 환자 안전 보장: 정확한 환자 정보 관리, 약물 상호작용 검증, 검사 결과 임계치 기반 알림 기능
4. 사용자 친화적 인터페이스: 의료진이 직관적으로 사용할 수 있는 현대적인 웹 인터페이스 구현
5. 확장 가능한 시스템 아키텍처: 모듈화된 구조로 향후 기능 추가 및 확장이 용이한 시스템 설계

**개발동기:**
기존 의료 환경에서 의료진이 직면하는 주요 문제점은 다음과 같습니다:
- 종이 기반 의무기록의 비효율성: 환자 정보 검색, 이력 조회에 시간이 많이 소요됨
- 반복적인 문서 작성 작업: 소견서, 처방전 등 반복적인 문서 작성으로 인한 업무 부담
- 의료 오류 위험: 약물 상호작용, 검사 결과 이상 수치를 놓칠 가능성
- 정보 통합의 어려움: 환자 정보, 검사 결과, 처방 내역 등이 분산되어 있어 통합 관리가 어려움

**필요성:**
최근 AI 기술의 발전과 함께 의료 분야에서도 AI 기반 보조 도구의 활용이 증가하고 있습니다. 특히 소규모 의료기관에서도 접근 가능한 비용으로 고급 EMR 기능을 제공하여 의료 격차 해소에 기여할 수 있는 시스템의 필요성이 대두되고 있습니다.

### 1-2 선행기술 조사내용, 기존 기술(제품)과의 차별성

#### 선행기술 조사

**국내외 EMR 시스템 시장 조사:**

국내외 EMR 시스템 시장을 조사한 결과, 대형 병원에서는 Epic, Cerner, Allscripts 등의 상용 EMR 시스템이 널리 사용되고 있습니다. 이러한 시스템들은 강력한 기능을 제공하지만, 높은 도입 비용과 복잡한 사용자 인터페이스로 인해 소규모 의료기관의 접근성이 낮은 실정입니다.

최근에는 오픈소스 EMR 시스템인 OpenEMR, GNU Health 등이 개발되어 있으나, 한국 의료 환경에 최적화되지 않았고, AI 기반 의료진 보조 기능이 부족한 한계가 있습니다. 또한 FHIR(Fast Healthcare Interoperability Resources) 표준을 활용한 의료 데이터 교환 표준이 확산되면서, 표준 기반 EMR 시스템의 필요성이 증가하고 있습니다.

**AI 기술의 의료 분야 적용 사례:**

AI 기술의 의료 분야 적용 사례를 조사한 결과, IBM Watson Health, Google Health AI 등이 의료진 보조 도구로 활용되고 있으며, 임상노트 요약, 진단 추천, 약물 상호작용 검사 등의 기능이 실제로 활용되고 있습니다.

**의약품 정보 및 상호작용 검사 시스템 조사:**

국내 의약품 정보 제공 시스템을 조사한 결과, 한국 식품의약품안전처에서 제공하는 DUR(의약품안전사용정보) OpenAPI가 있으며, 이를 통해 의약품 정보 및 상호작용 정보를 조회할 수 있습니다. 또한 의약품개요정보 API를 통해 추가적인 의약품 정보를 제공받을 수 있습니다. 이러한 공공 API를 활용하여 의료기관에서 활용 가능한 약물 데이터베이스를 구축할 수 있습니다.

**특허정보검색 결과:**

특허정보검색 사이트(kipris.or.kr)에서 "전자 의무기록", "EMR", "AI 의료진 보조" 등의 키워드로 검색한 결과, 유사한 기술 분야의 특허들이 존재합니다. 주요 특허들은 다음과 같습니다:

- 전자 의무기록 시스템 관련 특허: 기존 특허들은 주로 데이터 저장 및 관리에 초점을 맞추고 있으며, AI 기반 자동화 기능은 제한적입니다.
- 의료진 보조 시스템 관련 특허: 일부 특허에서 AI를 활용한 진단 보조 기능을 제안하고 있으나, 다중 AI 모델 통합 및 자동 폴백 메커니즘은 포함되지 않았습니다.

**사업화 기술 조사:**

네이버, 구글 등에서 "EMR 시스템", "AI 의료진 보조", "전자 의무기록" 등의 키워드로 검색한 결과, 다음과 같은 상용 제품 및 서비스가 확인되었습니다:

- **Epic Systems**: 대형 병원용 EMR 시스템으로 강력한 기능을 제공하나, 높은 도입 비용과 복잡한 인터페이스가 단점입니다.
- **Cerner**: 글로벌 EMR 솔루션으로 널리 사용되나, 소규모 의료기관에는 부적합합니다.
- **OpenEMR**: 오픈소스 EMR 시스템이지만, 한국 의료 환경에 최적화되지 않았고 AI 기능이 부족합니다.

#### 기존 기술(제품)과의 차별성

본 프로젝트의 주요 차별점은 다음과 같습니다:

1. **다중 AI 모델 통합 및 자동 폴백 메커니즘**: 여러 AI 모델을 통합하여 한 모델 실패 시 자동으로 다른 모델로 전환하는 안정적인 AI 서비스 제공
2. **공공 API 기반 약물 데이터베이스 구축**: 한국 식품의약품안전처 DUR API를 활용하여 실제 의약품 정보 및 상호작용 데이터를 기반으로 한 약물 상호작용 검사 시스템 구현
3. **실시간 통합 대시보드**: 금일 대기 환자 목록, 병원 일정, 처방/오더 보드, 캘린더 등을 한 화면에서 통합 관리
4. **검사 결과 자동 분석 및 시각화**: 바이탈 사인 및 검사 결과의 임계치를 자동으로 감지하여 이상 수치를 시각적으로 표시
5. **한국 의료 환경 최적화**: 한국 의료진의 업무 흐름에 맞춘 인터페이스 및 기능 설계, SOAP 형식 진료 기록 지원
6. **접근 가능한 비용 구조**: 오픈소스 기반으로 구축하여 소규모 의료기관도 접근 가능한 비용으로 고급 기능 제공

## 2. 과제의 해결방안 및 수행과정

### 2-1 과제의 해결방안

기존 의료 환경의 문제점을 해결하기 위해 다음과 같은 시스템을 설계했습니다:

1. **통합 대시보드**: 의료진이 한 화면에서 모든 업무를 관리할 수 있는 대시보드
2. **AI 기반 자동화**: 임상노트 요약, 증상 분석, 약물 상호작용 검사 등 AI 기능으로 업무 효율성 향상
3. **실시간 알림 시스템**: 검사 결과 이상 수치, 상급병원 이송 필요 등의 중요한 정보를 실시간으로 알림
4. **직관적인 사용자 인터페이스**: 의료진이 쉽게 사용할 수 있는 현대적인 웹 인터페이스

### 2-2 수행과정

**개발 단계별 수행 과정:**

**1단계: 기반 구조 설정 (1주차)**
- 프로젝트 초기 설정 및 개발 환경 구축
- Git 저장소 생성 및 협업 환경 구성
- 기술 스택 선정 및 아키텍처 설계
- 데이터베이스 스키마 설계 (Prisma)

**2단계: 백엔드 API 개발 (2-3주차)**
- Express.js 서버 구축 및 기본 라우팅 설정
- Prisma를 통한 데이터베이스 연동
- 환자 CRUD API 개발
- 관찰치(Observation) 관리 API 개발
- 문서 관리 시스템 개발 (소견서, 처방전 등 자동 생성)
- 검사 관리 시스템 개발
- 약물 데이터베이스 구축 (한국 식품의약품안전처 DUR API 활용)
- 약물 상호작용 검사 시스템 개발
- 처방 관리 시스템 개발 (약물 상호작용 검사 포함)

**3단계: AI Gateway 개발 (3-4주차)**
- 다중 AI 모델 통합
- 자동 폴백 메커니즘 구현
- Rate Limit 관리 시스템 구현
- 임상노트 요약 API 개발
- 증상 분석 API 개발
- 처방 가이드 API 개발
- 검사 결과 분석 API 개발

**4단계: 프론트엔드 UI 개발 (4-6주차)**
- 공통 컴포넌트 개발 (Button, Input, Card 등)
- 레이아웃 컴포넌트 개발 (Header, Sidebar)
- 대시보드 페이지 개발
- 환자 등록 모달 개발
- 진료 차트 모달 개발
- 검사 관리 페이지 개발
- 문서 관리 페이지 개발
- API 연동 및 상태 관리 구현

**5단계: 통합 및 테스트 (7-8주차)**
- 프론트엔드-백엔드 통합 테스트
- AI Gateway 연동 테스트
- 시뮬레이션 데이터 생성 및 테스트
- 버그 수정 및 성능 최적화
- 사용자 인터페이스 개선

**발생한 문제 및 해결 사례:**

1. **문제: AI 모델 API 호출 실패 시 전체 시스템 영향**
   - 해결: 다중 AI 모델 지원 및 자동 폴백 메커니즘 구현으로 한 모델 실패 시 다른 모델로 자동 전환

2. **문제: 대량 환자 데이터 처리 시 성능 저하**
   - 해결: 데이터베이스 인덱싱 및 페이지네이션 구현으로 검색 성능 개선

3. **문제: 복잡한 상태 관리로 인한 버그 발생**
   - 해결: Context API를 통한 중앙화된 상태 관리 및 타입 안정성을 위한 TypeScript 활용

4. **문제: 검사 결과 이상 수치 감지 로직의 복잡성**
   - 해결: 임계치 기반 플래그 시스템을 구현하여 자동으로 이상 수치 감지 및 시각적 표시

5. **문제: 문서 생성 시 템플릿 관리의 어려움**
   - 해결: 모듈화된 템플릿 시스템을 구축하여 각 문서 유형별로 독립적인 템플릿 관리

6. **문제: 약물 상호작용 검사를 위한 신뢰할 수 있는 데이터 소스 확보**
   - 해결: 한국 식품의약품안전처의 DUR API를 활용하여 공식적인 의약품 정보 및 상호작용 데이터를 수집하고, 상호작용 매트릭스를 구축하여 자동 검사 시스템 구현

## 3. 과제의 내용

### 3-1 과제의 이론적·기술적 근거

**이론적 근거:**

본 프로젝트는 다음과 같은 이론적 배경을 바탕으로 설계되었습니다:

1. **SOAP 형식 진료 기록**: 주관적 증상(Subjective), 객관적 소견(Objective), 평가(Assessment), 계획(Plan) 형식의 표준화된 진료 기록 방식
2. **FHIR 표준**: Fast Healthcare Interoperability Resources 표준을 참고한 의료 데이터 구조 설계
3. **의료 정보학 원칙**: 환자 안전, 데이터 무결성, 접근성 등의 의료 정보학 원칙 준수

4. **의료정보 표준**: 
   - SOAP 형식 진료 기록: 주관적 증상, 객관적 소견, 평가, 계획으로 구성된 표준화된 진료 기록
   - FHIR 표준 참고: Fast Healthcare Interoperability Resources 표준을 참고한 의료 데이터 구조 설계
   - 의약품안전사용정보(DUR) 활용: 한국 식품의약품안전처의 DUR API를 활용한 약물 정보 및 상호작용 검사

**기술적 근거:**

1. **3-Tier 아키텍처**: 프레젠테이션 계층(프론트엔드), 비즈니스 로직 계층(백엔드), 데이터 계층(데이터베이스)으로 분리하여 확장성과 유지보수성 향상
2. **RESTful API 설계**: 표준 HTTP 메서드를 활용한 일관된 API 인터페이스
3. **컴포넌트 기반 개발**: React의 컴포넌트 기반 아키텍처를 활용한 재사용 가능한 UI 구성
4. **다중 AI 모델 통합**: 여러 AI 모델을 통합하여 안정성과 성능 향상

### 3-2 과제의 개념설계, 상세설계 및 계산

**시스템 아키텍처:**

본 시스템은 3-Tier 아키텍처로 구성되었습니다:

1. **프론트엔드 (Frontend)**: React 18 + TypeScript 기반의 사용자 인터페이스
   - 컴포넌트 기반 모듈화 설계
   - Context API를 통한 전역 상태 관리
   - 반응형 레이아웃 (Grid, Flexbox)

2. **백엔드 (Backend)**: Node.js + Express.js 기반의 RESTful API 서버
   - Prisma ORM을 통한 데이터베이스 관리
   - 환자, 진료기록, 검사, 처방 등 핵심 데이터 관리
   - 문서 자동 생성 시스템

3. **AI Gateway**: 다중 AI 모델 통합 게이트웨이
   - 여러 AI 모델 지원
   - 자동 폴백 메커니즘
   - Rate Limit 관리

**주요 기능 모듈:**

1. **환자 관리 모듈**: 신규 환자 등록, 환자 검색, 이력 조회
2. **진료 차트 모듈**: SOAP 형식 진료 기록 작성, AI 자동 요약
3. **검사 관리 모듈**: 검사 오더 생성, 결과 입력, 이상 수치 자동 감지
4. **처방 관리 모듈**: 처방전 생성, 약물 상호작용 검사
5. **문서 관리 모듈**: 소견서, 진료 보고서, 처방전 자동 생성
6. **대시보드 모듈**: 금일 대기 환자, 일정, 알림 통합 관리

**데이터베이스 설계:**

Prisma ORM을 사용하여 다음과 같은 주요 엔티티를 설계했습니다:

- **User**: 의료진 사용자 정보
- **Patient**: 환자 기본 정보 (이름, 생년월일, 전화번호 등)
- **Encounter**: 진료 방문 기록
- **Observation**: 관찰치 (바이탈 사인, 검사 결과 등)
- **Prescription**: 처방 정보
- **Document**: 생성된 문서 정보

**검사 결과 이상 수치 감지 로직:**

각 검사 항목별로 정상 범위를 정의하고, 입력된 검사 결과가 정상 범위를 벗어나는 경우 자동으로 플래그를 설정합니다. 예를 들어:

- WBC (백혈구): 정상 범위 4,000-10,000/µL, 12,500/µL 입력 시 비정상 플래그 설정
- 체온: 정상 범위 36.5-37.5°C, 38.5°C 입력 시 비정상 플래그 설정

**약물 데이터베이스 및 상호작용 검사 시스템:**

본 시스템은 한국 식품의약품안전처의 DUR(의약품안전사용정보) OpenAPI를 활용하여 약물 데이터베이스를 구축했습니다:

1. **약물 데이터 수집**: 
   - data.go.kr의 DUR API를 활용하여 의약품 정보 수집
   - 의약품개요정보 API를 통한 추가 정보 통합
   - 약물명, 성분명, 카테고리, 상호작용 정보 등 구조화된 데이터 구축

2. **약물 상호작용 검사 로직**:
   - 약물 쌍에 대한 상호작용 매트릭스 구축
   - 처방 시 여러 약물 간 상호작용 자동 검사
   - 상호작용 심각도 판정 (low, medium, high, critical)
   - 금기사항 및 주의사항 자동 검색 및 알림

3. **처방 가이드 생성**:
   - 환자 특성(연령, 기저질환 등)에 따른 용량 조절 권고
   - 약물별 부작용 및 주의사항 제공
   - 고령 환자, 신장질환 환자 등 특수 상황별 맞춤 가이드

**다중 AI 모델 통합 방식:**

본 시스템은 여러 AI 모델을 통합하여 안정성과 성능을 향상시켰습니다:

1. **AI Gateway 아키텍처**: 
   - 여러 AI 모델을 통합 관리하는 게이트웨이 서버 구축
   - 모델별 API 호출 인터페이스 표준화
   - Rate Limit 관리 및 요청 큐잉 시스템

2. **자동 폴백 메커니즘**:
   - 한 모델의 API 호출 실패 시 자동으로 다른 모델로 전환
   - 모델별 응답 시간 모니터링 및 최적 모델 선택
   - 시스템 안정성 및 가용성 향상

3. **AI 기능 활용**:
   - 임상노트 자동 요약: SOAP 형식 진료 기록을 AI를 활용하여 자동 요약
   - 증상 분석: 환자 증상을 분석하여 진단 추천 제공
   - 처방 가이드: 약물 처방 시 AI 기반 처방 가이드 생성
   - 검사 결과 분석: 검사 결과를 분석하여 이상 수치 감지 및 해석 제공
