# 세부 지도내용

> 팀별 역할·저장소 구조·API 연동 담당 등 **최신 실행 기준**은 저장소 루트 **`README.md`** 를 함께 참고한다.  
> 역할 요약: **김지현**(팀장, AI, Backend) — Express·업무 모듈·**AI Gateway** / **오수민**(AI, Frontend) — React 전반·대시보드·`exam-flow`·`api` / **송유찬**(Backend, DataBase) — **Prisma**·생성기·**`drugDatabase`**·`DataBase/`  
> **프로젝트 수행 기간:** 2025년 3월 1일 ~ 2025년 5월 30일 (아래 지도일은 해당 기간 내 금요일 미팅 기준)

## 2025년 3월 13일 (금요일)
**지도 내용:**
- 프로젝트 주제 선정 및 프로그램 구상에 대한 지도
- EMR(전자 의무기록) 시스템 개발의 필요성 및 목표 설정
- **외래 진료** 흐름(대기→검사→재진)을 한눈에 보이게 할지, 입원까지 포함할지 등 **프로젝트 범위** 논의 → 본 작품은 **외래 중심·입원 제외** 방향 정리
- 프로젝트 범위 및 핵심 기능 정의
- 팀원별 역할 분담 방안 논의  
  - **김지현**: 백엔드 API·비즈니스 로직·AI Gateway(다중 LLM) 총괄  
  - **오수민**: 프론트엔드 UI/UX, 대시보드·전용 화면, 프론트 `api`·AI 화면 연동  
  - **송유찬**: DB 스키마·마이그레이션, 시드·샘플 데이터, 약물 JSON·`drugDatabase`  
- 프로젝트 일정 수립 및 마일스톤 설정

## 2025년 3월 27일 (금요일)
**지도 내용:**
- 시스템 아키텍처 설계 방향에 대한 지도(프론트 / 백엔드 / **AI Gateway** / DB·데이터 자산 분리)
- 사용할 개발도구 및 기술 스택 추천
  - 프론트엔드: React 18 + TypeScript, Vite
  - 백엔드: Node.js + Express.js, Prisma ORM
  - 데이터베이스: SQLite (개발용)
  - **AI Gateway**: OpenAI, Anthropic, Google 등 **다중 모델**·폴백·Rate Limit
  - 약물·샘플 데이터: `DataBase/` 스크립트·JSON, 백엔드 `drugDatabase` 연계 방안
- 개발 환경 구축 방법 안내
- Git 버전 관리 및 협업 프로세스 설정

## 2025년 4월 10일 (금요일)
**지도 내용:**
- 데이터베이스 스키마 설계 지도(**송유찬** 주도, **김지현**이 앱에서 Prisma 클라이언트 활용)
- Prisma ORM을 활용한 데이터 모델링 방법
- RESTful API 설계 원칙 및 엔드포인트 구조(**김지현** Express 라우트·업무 모듈)
- 프론트엔드-백엔드 통신 구조 설계(**오수민** `frontend/src/api/*` 와의 맞춤)
- 컴포넌트 기반 UI 설계 방법론

## 2025년 4월 24일 (금요일)
**지도 내용:**
- 시스템 구현 방법에 대한 구체적 지도
- 환자 관리 모듈 구현 방향
- AI Gateway 통합 방법 및 다중 모델 폴백 메커니즘 구현
- **홈 대시보드**: 사용자 피드백 반영 시 **기능 중복 제거**, **3-Column(대기·검사·일정/알림)**·**조회 중심(view-only)** 정책 검토
- **검사 관리 전용 화면**: 일정·매트릭스·운영 요약 등 확장 UI(`exam-flow` 등) 구현 방향
- 검사 결과 이상 수치 자동 감지 로직 설계
- 문서 자동 생성 시스템 구현 방법

- **API 연동**: 환자·처방·약물·관찰치 등에서 **프론트 모듈(오수민)** 과 **REST(김지현)**·**스키마·데이터(송유찬)** 가 어긋나지 않도록 계약(엔드포인트·타입) 점검
